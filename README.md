# Vibe Coding Skills

**AI 輔助開發的結構化基礎框架**

> 透過清晰的上下文、原子化任務、記憶系統與無害工程原則，最大化 AI Coding Assistant 的效能。

---

## 為什麼需要這個框架？

AI Coding Assistant（Claude Code、Cursor、GitHub Copilot 等）能大幅加速開發，但前提是──**你給對上下文**。

沒有結構化的指引，AI 生成的程式碼容易出現：
- 風格不一致，難以維護
- 缺少測試，回歸風險高
- 無視安全邊界，引入漏洞
- 每次對話從零開始，浪費大量時間建立上下文

**這個框架就是為了解決這些問題。**

## 快速入門

### 1. 從這裡開始

```bash
# 閱讀核心文件（建議順序）
1. README.md     ← 你在這裡
2. PRINCIPLES.md ← 了解無害工程原則
3. CONTEXT.md    ← 填寫你的專案上下文
4. WORKFLOW.md   ← 學會開發循環
5. SKILLS.md     ← 掌握 AI 提示詞模式
```

### 2. 每次開發對話這樣做

```markdown
# 與 AI 開始對話時，先載入上下文：

我正在開發 {{專案名稱}}。

## 專案願景
{{CONTEXT.md → 核心目標與邊界}}

## 目前狀態
{{CONTEXT.md → 技術棧與專案結構}}

## 今天的任務
{{從 WORKFLOW.md 選擇對應工作流程}}
```

### 3. 每天結束時

```markdown
# 更新日誌，保持 AI 的記憶連續
- 更新 logs/implementation-log.md（今天做了什麼）
- 更新 logs/decisions-log.md（做了什麼重要決策）
- 記錄任何新發現的 Bug 到 logs/bug-log.md
```

## 適用對象

- 使用任何 AI Coding Assistant 的開發者（Claude Code、Cursor、GitHub Copilot、Windsurf 等）
- 個人開發者或小團隊
- 從零開始的新專案，或既有專案導入 AI 輔助開發
- 想要在速度與品質之間取得平衡的任何人

---

## 核心理念：Harmless Engineering

**速度不該以品質為代價。AI 是副駕駛，不是自動駕駛。**

詳見 [PRINCIPLES.md](./PRINCIPLES.md)，了解如何讓 AI 輔助開發既快速又安全。

---

> *「Vibe Coding 不是放手讓 AI 亂寫，而是給 AI 夠好的上下文，讓它寫出你真正想要的程式碼。」*
